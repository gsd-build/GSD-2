## What Was Built

### Task 1 — useSessionManager.ts: Fork + Stuck Detection

**Session fork message copy:**
- Added `pendingForkRef = useRef<string | null>(null)` to track which session is being forked
- `createSessionAction` now sets `pendingForkRef.current = forkFrom` before sending `session_create`
- In `session_update` handler: after `applyState(updated)`, if `pendingForkRef.current` is set, find the new session (present in `tabs` but not in `stateRef.current.sessions` pre-update) and copy parent messages via `setMessagesBySession`

**Stuck session detection after reconnect:**
- Added `stuckSessionId` state (null when no stuck session)
- Added `stuckTimerRef` with 3000ms timeout — fires `setStuckSessionId(processingSession.id)`
- Added `isReconnectedRef` — set to `true` in `onReconnect`, consumed (set to false) in `session_update`
- In `chat_event` handler: clears `stuckTimerRef` and calls `setStuckSessionId(null)` on any activity
- Added `reconnectSession` callback: clears stuck state, re-sends `session_list` request
- Cleanup `useEffect` removes stuckTimerRef on unmount

**Interface changes:**
- `UseSessionManagerResult` gains `stuckSessionId: string | null` and `reconnectSession: (sessionId: string) => void`

### Task 2 — Reconnect Banner UI + Tests

**SingleColumnView.tsx:**
- Added `stuckSessionId?: string | null` and `onReconnectSession?: (sessionId: string) => void` props
- Renders amber banner with `role="alert"` inside chat view when `stuckSessionId` is set
- Banner text: "GSD may still be running. Reconnect to attach to the session."
- Reconnect button in amber (#F59E0B background, #0F1419 text) calls `onReconnectSession?.(stuckSessionId)`

**AppShell.tsx:**
- Destructures `stuckSessionId` and `reconnectSession` from `useSessionManager`
