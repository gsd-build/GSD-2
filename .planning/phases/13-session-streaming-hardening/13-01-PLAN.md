  <behavior>
Pi SDK outputs NDJSON. Each line is a JSON object. classifyPiSdkEvent(raw) inspects the shape and returns a typed GSD2StreamEvent or null.

**GSD2EventType variants and detection rules:**

- `plain_text` — `{ type: "text", text: string }` — plain assistant text
- `tool_use` — `{ type: "tool_use", name: string, input: unknown }` — tool invocation
- `tool_result` — `{ type: "tool_result", tool_use_id: string, content: unknown }` — tool output
- `phase_transition` — `{ type: "phase_transition", phase: "Research" | "Planning" | "Executing" | "Complete" }` — auto mode phase change
- `cost_update` — `{ type: "cost_update", total_cost_usd: number, input_tokens: number, output_tokens: number }` — running cost
- `stuck_detection` — `{ type: "stuck_detection", message: string }` — loop warning
- `timeout` — `{ type: "timeout", message: string, elapsed_seconds: number }` — timeout warning
- `auto_mode_announcement` — `{ type: "auto_mode_announcement", mode: "start" | "stop", slice?: string }` — auto mode lifecycle

**Classification behavior:**
- Input not an object → null
- Input has unrecognised type string → null (logged as unknown, never throws)
- Input missing required field for its type → null (graceful degradation)
- All 8 variants match exactly

**TDD cases (write these tests first, run — all must FAIL, then implement):**

- classifyPiSdkEvent({ type: "tool_result", tool_use_id: "tu_1", content: "ok" }) → { kind: "tool_result", tool_use_id: "tu_1", content: "ok" }

- classifyPiSdkEvent({ type: "stuck_detection", message: "Looping" }) → { kind: "stuck_detection", message: "Looping" }
- classifyPiSdkEvent({ type: "timeout", message: "Timed out", elapsed_seconds: 120 }) → { kind: "timeout", message: "Timed out", elapsed_seconds: 120 }

  output_tokens: number;

  | { kind: "plain_text"; text: string }

  | { kind: "phase_transition"; phase: PhaseTransitionPhase }
