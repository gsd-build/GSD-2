<objective>
Wire the WebSocket reconnect to send a 'refresh' signal on reconnect (triggering full state re-derive), and surface crash recovery UI in ChatView.

Purpose: The backoff reconnect mechanism already exists. This plan closes two gaps: (1) on reconnect the client must explicitly request a fresh state snapshot from disk, not rely on whatever the server has in memory; (2) when gsd crashes, the UI must show a recovery banner rather than silently staying stuck.

Output:
- useReconnectingWebSocket sends 'refresh' on reconnect (not on first connect)
- ChatView renders a crash recovery banner when isCrashed prop is true
- Reconnect button in banner clears crash state and sends new 'refresh' message
</objective>
