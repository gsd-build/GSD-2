## Accomplishments

- Added `wired: boolean` to `SessionState` and a guard in `wireSessionEvents` — calling it twice on the same session no longer duplicates event handlers silently
- Changed `reconcileInterval` to `let` and added `clearInterval` before / `setInterval` after the switch body in `switchProject` — eliminates the stale-closure race condition during project switches
- Moved `MAX_SESSIONS = 4` to a single `export const` in `types.ts`; removed local duplicates from `session-manager.ts` and `SessionTabs.tsx`
- Added `validateConfigState()` to `state-deriver.ts` replacing the unsafe `configRaw as ConfigState` cast — malformed or partially-written `config.json` returns `DEFAULT_CONFIG_STATE` values for invalid fields

## Decisions Made

- `wired` flag lives on `SessionState` rather than a separate `Set<string>` in `pipeline.ts` — keeps the guard state co-located with the session object and avoids an extra closure variable
- `validateConfigState` uses `typeof` field-by-field checks rather than Zod — no new dependencies; consistent with the existing pattern across the codebase (RESEARCH.md anti-pattern warning respected)
- `reconcileInterval` restart is inline in `switchProject` using the same body — minimal diff, avoids introducing a named function just for this case
